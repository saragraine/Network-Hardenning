# 30 / 60 / 90 Day Plan

## 30 days
- Normalize file naming for evidence and configs
- Remove temporary or duplicate configuration fragments
- Review patch level on all lab systems
- Rotate SSH keys used for administration
- Verify backups of critical configuration files

## 60 days
- Improve segmentation clarity and rule minimization
- Tighten remote-access policy enforcement
- Review TLS configuration against a fixed baseline
- Tune IDS rules to reduce noise and improve signal quality
- Evaluate additional edge protections such as WAF-style controls where relevant

## 90 days
- Move key configs into infrastructure-as-code where possible
- Add CI checks for regression scripts and shell syntax
- Establish periodic policy and evidence review
- Add automated collection of key telemetry artifacts
- Formalize ownership for firewall, TLS, IDS, and remote-access changes
